Reconocimiento de Expresiones Faciales con Redes Neuronales

Exploramos el dataset FER2013, iteramos sobre arquitecturas CNN y aplicamos transfer learning con ResNet18 para clasificar siete emociones humanas.

Mayo 2026 PyTorch · CNN · Transfer Learning 10 min de lectura
01 — Planteamiento

El problema: enseñar a una red a leer emociones

El reconocimiento automático de expresiones faciales es una tarea con aplicaciones que van desde interfaces más empáticas hasta sistemas de salud mental asistidos por IA. Sin embargo, traducir los matices del gesto humano en una etiqueta discreta sigue siendo un problema difícil: las emociones se superponen, la iluminación varía y los rostros son geométricamente muy distintos entre personas.

En este proyecto nos propusimos clasificar imágenes de rostros en siete categorías emocionales usando redes neuronales convolucionales. El objetivo no era solo maximizar la exactitud, sino entender qué decisiones de diseño mueven la aguja de forma significativa y por qué.

Pregunta central: ¿Cuánto aportan el aumento de datos, la normalización por lote y el transfer learning frente a una CNN base entrenada desde cero?

02 — Dataset

Análisis exploratorio: FER2013

Trabajamos con FER2013 (Facial Expression Recognition 2013), un dataset estándar en la literatura que contiene imágenes en escala de grises de 48 × 48 píxeles, etiquetadas en siete clases: enojo, asco, miedo, felicidad, tristeza, sorpresa y neutral. El conjunto de entrenamiento tiene aproximadamente 28 000 muestras.

03 — Decisiones de Diseño

Qué decidimos y por qué

Iteramos a través de cuatro configuraciones principales, cada una introduciendo un cambio deliberado sobre la anterior para aislar el efecto de cada decisión. A continuación las decisiones de diseño más relevantes:

⟳

Data Augmentation

Volteos horizontales aleatorios, rotaciones de hasta ±15° y variaciones de brillo para ampliar el espacio de hipótesis del modelo.

◫

Batch Normalization

Insertar BN después de cada bloque convolucional estabilizó el entrenamiento y nos permitió usar tasas de aprendizaje más altas sin divergencia.

◈

Arquitectura CNN propia

Tres bloques conv–pool progresivos (32→64→128 filtros) con activación ReLU y un clasificador lineal al final.

↗

Transfer Learning ResNet18

Fine-tuning de ResNet18 preentrenada en ImageNet con imágenes redimensionadas a 224×224 y la capa final reemplazada para siete clases.

Arquitectura CNN base

La red propia fue diseñada deliberadamente simple: sin skip connections ni bloques de atención. Queríamos una línea base clara antes de introducir complejidad. El esquema general fue: 

Input 48×48×1
  └─ Conv(32) → BN → ReLU → MaxPool
  └─ Conv(64) → BN → ReLU → MaxPool
  └─ Conv(128) → BN → ReLU → MaxPool
  └─ Flatten → Dropout(0.4) → Linear(128) → Linear(7)

Por qué ResNet18 y no algo mayor

Consideramos ResNet50 y VGG16, pero el tamaño reducido del dataset (≈28k imágenes) hacía probable el sobreajuste con arquitecturas más grandes. ResNet18 ofrece residual connections suficientes para capturar texturas y estructuras faciales, sin la capacidad extra que requeriría muchos más datos para generalizarse bien.

04 — Resultados

Comparativa de modelos

Evaluamos cada configuración sobre el conjunto de prueba oficial de FER2013. La exactitud reportada es top-1 en las siete clases. Encontramos que cada cambio introducido mejoró el rendimiento de forma consistente, aunque la mayor ganancia individual vino del transfer learning.

1 CNN base No No 52.0 %
2 CNN + Augmentation v1 Básica No 63.0 %
3 CNN + Augmentation v2 Extendida Sí 64.4 %
4 ResNet18 224×224 Mejor Extendida Sí (nativa) 66.8 %

El salto de 11 puntos porcentuales entre la CNN base y la primera versión con augmentation fue el más pronunciado de todos, lo que confirma que el aumento de datos es una inversión de alta rentabilidad cuando el dataset es limitado. El paso a ResNet18 añadió otros 2.4 puntos gracias a los pesos preentrenados en millones de imágenes naturales, que proveen representaciones de bajo nivel (bordes, texturas) difíciles de aprender con tan pocas muestras.

Nota sobre el techo del dataset: La literatura reciente reporta ~73–75 % en FER2013 con arquitecturas grandes y datos adicionales. Nuestro 66.8 % con un modelo ligero y sin datos externos es un resultado competitivo para el alcance de este proyecto.

05 — Conclusiones

Qué aprendimos y hacia dónde va esto

El ejercicio demostró que las ganancias de rendimiento en este problema no vienen de escalar la arquitectura, sino de intervenir en el régimen de entrenamiento. El augmentation redujo el sobreajuste de manera más efectiva que añadir capas; la normalización por lote aceleró la convergencia y permitió explorar hiperparámetros más agresivos; y el transfer learning compensó la escasez de datos con conocimiento visual preexistente.

También encontramos que el desbalance de clases sigue siendo el obstáculo más subestimado del dataset: las métricas globales ocultan un rendimiento pobre en clases como Asco y Miedo. Cualquier mejora futura debería medirse también con F1 por clase, no solo con exactitud global.

Trabajo futuro

  • Aplicar class-weighted loss o oversampling para atacar el desbalance de Asco y Miedo.
  • Explorar AffectNet o RAF-DB, datasets con mayor diversidad y menos ruido que FER2013.
  • Probar MobileNetV3 como alternativa eficiente a ResNet18 para despliegue en dispositivos de borde.
  • Implementar Grad-CAM para visualizar qué regiones faciales activan cada clase y detectar sesgos.
  • Evaluar robustez ante variaciones de pose utilizando landmarks faciales como preprocesado.
